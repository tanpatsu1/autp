# Token Rules

## 実装単位
- 1 タスク 1 目的に絞る
- `現状 / 目的 / 触るファイル / 完了条件 / 懸念点` の 5 項目で整理する

## 読み込み順
- 最初に読むのは `docs/current-status.md`
- 追加で読む repo ファイルは必要最小限にする
- 全読みに行く前に `rg` と関連箇所確認で範囲を絞る

## 出力フォーマット
- 現状整理
- 実施内容
- 検証結果

## 注意
- 目的外の変更は入れない
- 長い説明より根拠と結論を優先
- 1 PR に複数テーマを混ぜない

## 実装後テンプレート
```text
現状:
目的:
触るファイル:
完了条件:
懸念点:
```
